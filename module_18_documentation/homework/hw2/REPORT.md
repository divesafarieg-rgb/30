## Результаты испытаний

| Обозначение | Описание                          |
|-------------|-----------------------------------|
| -O          | Без дополнительной настройки      |
| +O          | С дополнительной настройкой       |
| -S          | Без использования сессии          |
| +S          | С использованием сессии           |
| -T          | Без использования многопоточности |
| +T          | С использованием многопоточности  |

### Выводы

1. **Многопоточность (+T)** даёт наибольший прирост производительности (в 2-9 раз).
2. **Сессии (+S)** практически не влияют на скорость при использовании многопоточности.
3. **Настройка сервера (+O)** не даёт значительного улучшения производительности.
4. **Рекомендуется использовать многопоточность** при большом количестве запросов.
